Security Model

Zero-Knowledge Guarantee

Worker never receives plaintext vulnerability details. All sensitive data is encrypted client-side before transmission.

How It Works

  • • Reporter browser encrypts using org public key (P-256 ECDH + HKDF + AES-GCM).
  • • Worker receives ciphertext package and forwards to org inbox.
  • • D1 database stores only domain, username (optional), hash, timestamp, and org public key.
  • • Only the organization with the private key can decrypt the vulnerability report.

Encryption Algorithm

• Key Exchange: ECDH P-256

• Key Derivation: HKDF-SHA256

• Encryption: AES-256-GCM

• Salt: 16 bytes random

• IV: 12 bytes random

What's Stored in Database

  • ✓ Domain name
  • ✓ BLT username (optional, for points)
  • ✓ Submission ID and timestamp
  • ✓ SHA-256 hash of encrypted package
  • ✗ Vulnerability details (encrypted, never stored)
  • ✗ Screenshots (encrypted, never stored)
